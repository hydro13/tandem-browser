# /vs/<competitor> page structure

All comparison pages follow this honest, humble structure.

**Tone rules:**
1. Never aggressive or dismissive — competitors serve real users.
2. Always include the dev-preview disclaimer banner.
3. "When to choose [competitor]" must be 4-5 substantive positive reasons.
4. No "Tandem wins everything" framing — when competitor has a real advantage, say so.

**Page sections in order:**
1. Title: "Tandem Browser vs <Competitor> — <neutral one-line descriptor>"
2. Meta description: factual and balanced
3. Article JSON-LD with both products as `about` entities
4. H1: "Tandem Browser vs <Competitor>"
5. Lead paragraph: neutral, acknowledges both serve real users
6. Preview-banner: "Tandem Browser is in active development preview..."
7. Side-by-side comparison table (8-12 rows; mix of Tandem-lean, competitor-lean, and neutral rows)
8. Two parallel "When to choose..." columns with 4-5 substantive bullets each
9. "What they have in common" block (positive, shared-mission framing)
10. Footer with link back to home and GitHub
